## Описание варианта
**Вариант 19. Задача: Полиномиальный хэш (window_hash)**  
Реализуйте функцию `window_hash(s, m)`, которая вычисляет полиномиальный хэш первых `m` символов строки `s` с основанием `31`.  
Формула:  
\[
H = \sum_{i=0}^{m-1} \text{ord}(s[i]) \cdot 31^{\,m-1-i}
\]  
Если `m <= 0`, возвращается `0`; если `m > len(s)`, хэш вычисляется для всей строки.

**Примеры тестов:**
- `window_hash("ABC", 3)` → `64578`  
  (проверка: `65*31² + 66*31 + 67 = 64578`)
- `window_hash("ABC", 2)` → `2081`  
  (проверка: `65*31 + 66 = 2081`)

## Принцип работы
Функция `window_hash` принимает строку `s` и целое число `m`. Она проверяет типы аргументов, затем обрабатывает граничные значения `m`. Далее в цикле для каждого символа из первых `m` (или из всей строки, если `m` больше её длины) вычисляется код символа (`ord`), умножается на соответствующую степень `31` и суммируется. Результат возвращается как целое число.

## Запрос данных
Скрипт запрашивает у пользователя ввод строки и числа `m` через консоль (`input()`). Ввод преобразуется: строка остаётся строкой, число приводится к целому типу.

## Вычисление
После получения данных вызывается функция `window_hash(s, m)`, и вычисленный хэш сохраняется в переменную.

## Вывод и проверка
На экран выводятся:
- Исходная строка `s`
- Значение `m`
- Полученный хэш
- Ожидаемый хэш (пользователь вводит его для проверки)

Затем скрипт сравнивает полученный и ожидаемый хэши. Если они равны, выводится **"верно"**, иначе — **"неверно"**.
